## üß© Current setup (on one laptop)

All machines run as virtual machines on a single laptop.

### Hosts

- **Windows 10 Pro (VM)**  
  - Role: target system, client, and server for generating traffic.  
  - Logs: Windows Event Logs (Security, System, Application).  
  - Tools: basic networking tools, browsers, test services.

- **Kali Linux (VM)**  
  - Role: attacker / tester machine.  
  - Tools: `nmap`, `tcpdump`, other scanning and exploitation tools.  
  - Traffic: generates scans and test attacks against Windows and Ubuntu.

- **Ubuntu Linux (VM)**  
  - Role: analysis and future Splunk host.  
  - Logs: system logs, auth logs, and any custom logs generated by scripts.  
  - Tools: Wireshark, tcpdump, basic SIEM‚Äëlike analysis scripts.

### Network layout (current)

All VMs are connected to the same internal virtual network (e.g., `VMnet2` in VirtualBox / VMware).  
- Each VM has a static or DHCP IP in the same subnet (e.g., `192.168.56.0/24`).  
- The laptop host is used only as the hypervisor; no services are exposed directly from it.
